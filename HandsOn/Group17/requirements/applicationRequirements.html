Project description

The aim of this application is to raise awareness of the current quality of the air in a big city as Madrid.

This application will show a map of Madrid with the location of the different stations for air quality measurement with a colour indicating the quality of the air according to the classification established by the ICA. In this App, the user will be able to check the quality of the air in his current location or in any location of the city. The application will show with detail the concrete values of the gases and the degree of risk associated at the area selected. Also, the user will be able to filter locations according to the quality of the air and to check the nearest locations with better air quality. Addicionally, every location will have available the data of the air quality evolution in the recent years.

Application Requirements

For the implementation of this App, we have considered the following requirements:

  • The format for our application will be Web App.
  • For the implementation and the treatment of data we use Java.
  • For UI development we will use HTML5 with bootstrap. In case we need any particular tool for the integration of data in a particular format we will add another technology.
  • The UI should be very visual and easy to understand. On the left side it will contain a search form to select certain location and a panel with the data about the current location of the user. On the right side, the application will show a map with visual and graphic information of all the stations of air measurement.

Data

According to the best practices for the process of generation of Linked Data, the data that we have selected for this application will satisfy the following requirements:

  • R1. Data are in one of the selected smart city domains.
  • Our application will be focused on Air quality.
  • R2. Data are available as a CSV file.
  • For this application we will use different CSV files related to Air quality and Streets in Madrid. All of the data will be extracted from CSV files, which may be accesed in the following links:
    • Calidad del aire. Datos en tiempo real: https://datos.madrid.es/sites/v/index.jsp?vgnextoid=41e01e007c9db410VgnVCM2000000c205a0aRCRD&vgnextchannel=374512b9ace9f310VgnVCM100000171f5a0aRCRD
    • Calidad del aire. Datos diarios años 2001 a 2019: https://datos.madrid.es/sites/v/index.jsp?vgnextoid=aecb88a7e2b73410VgnVCM2000000c205a0aRCRD&vgnextchannel=374512b9ace9f310VgnVCM100000171f5a0aRCRD
    • Callejero Oficial del Ayuntamiento de Madrid: https://datos.madrid.es/sites/v/index.jsp?vgnextoid=b3c41f3cf6a6c410VgnVCM2000000c205a0aRCRD&vgnextchannel=374512b9ace9f310VgnVCM100000171f5a0aRCRD
  • R3. Data have an open license so they can be published.
  • Open distribution license for commercial and non-commercial use.
  • R4. Data can be easily linked with generic real-world entities (e.g., locations).
  • Our data can be easily linked to locations in the real world. As we have data of quality air in different zones, we may relate concrete locations belonging to those zones (like streets or addreses) to a certain classification of quality air.
  • R5. Documentation exists for the data (optional).
  • Every one of the CSV files come with at least one official document associated. In these documents we may find information about the data or the representation of data in the file.
  • R6. Data can come from multiple data sources (optional).
  • All of our datasets are extracted from: https://datos.madrid.es/portal/site/egob .
