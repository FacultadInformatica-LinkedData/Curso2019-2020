Análisis de los datos:

Para cada fuente tenemos los siguientes datos:

Field Type Comments
Código de Fuente String Hay algunas fuentes con el mismo identificador (28 fuentes), pero en diferentes lugares o distritos. Tampoco hay elementos nulos o vacios
Zona Verde/Via Publica String el String sólo puede tomar dos valores: ZV (Zona verde) o VP (Vía publica). No hay elementos nulos o con error, tampoco vacíos.
Distrito String Nombre del distrito donde está la fuente, hay muchos valores duplicados. No hay elementos nulos o con error, tampoco vacíos.
Dirección String En algunos casos la dirección incluye un numero, y hay bastantes duplicados (829). No hay elementos nulos o con error, tampoco vacíos.
Complemento Dirección String En algunos casos la dirección incluye un numero, y hay bastantes duplicados (815). Hay 551 elementos nulos o vacios.
Coordenada-X String Rango [429544,68641-452246,27961], indican la coordenada-x de la localización hay algún elemento duplicado (16). No hay elementos nulos o con error, tampoco vacíos.
Coordenada-Y String Rango [4464487,999-4493789,807], indican la coordenada-y de la localización, hay algún elemento duplicado (18). No hay elementos nulos o con error, tampoco vacíos.
Sistema de coordenadas String Inidcan el sistemas de coordenadas, en todos los elementos es el mismo: "ETRS89 / UTM Zona 30N". No hay elementos nulos o con error, tampoco vacíos.
Estado String Indican el estado del servicio, puede tomar los valores: "EN SERVICIO" o "FUERA DE SERVICIO" (112). No hay elementos nulos o con error, tampoco vacíos.
Observaciones String Insica si hay algun dato relevante que añadir que no esté incluido en los campos del dataser, la mayoría tienen este campo vacío/null, y puede tener los valores encontrados son: "DEFICIENCIA ESTRUCTURAL", "NO APTA CONSUMO" y "VANDALISMO REITERADO". No hay elementos con error.
Fecha datos String Indica la fecha de recogida de los datos, es igual en todos los elementos y toma el valor "1-ago.-19". No hay elementos nulos o con error, tampoco vacíos.
